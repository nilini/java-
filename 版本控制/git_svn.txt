1、Git和SVN的区别
    git                          svn
    分布式                        集中式
2、分布式和集中式
    SVN是集中式版本控制，必须要有一个中央服务器，中央服务器崩了则无法工作。
    Git是分布式版本控制，没有中央服务器，每个人的电脑都是一个完整的版本库，可以相互拉取推送。但为了方便会有一台充当“中央服务器”的电脑，方便大家交换修改。
3、常见命令
    本地目录为工作区
    git add 把文件修改添加到暂存区。
    git commit 把暂存区的所有内容提交到当前分支
    git push
    git log 
        穿梭前，用git log可以查看提交历史，以便确定要回退到哪个版本。
        要重返未来，用git reflog查看命令历史，以便确定要回到未来的哪个版本。
    git reset –hard 版本号
    git diff 版本号码1 版本号码2
    git checkout --<file> 从先从缓存区中拉取版本还原，如果没有再到版本库中拉取还原。
    git rm --cached "文件路径"，不删除物理文件，仅将该文件从缓存中删除
    git rm --f  "文件路径"，不仅将该文件从缓存中删除，还会将物理文件删除（不会回收到垃圾桶）
